# <a name="delivery-optimization-settings-for-intune"></a>Paramètres d’optimisation de livraison pour Intune

Cet article répertorie les paramètres d’optimisation de livraison Intune prend en charge pour les appareils qui exécutent Windows 10 ou version ultérieure.  

La plupart des options dans la console Intune mappent directement aux paramètres d’optimisation de livraison sont examinées en détail dans la documentation de Windows, pour lesquelles des liens vers des contenus pertinents sont fournies.  Paramètres ou options qui sont spécifiques à Intune ne contiennent pas de liens vers du contenu supplémentaire.  

Les tableaux suivants sont les suivantes :  

- **Paramètre**: le paramètre tel qu’il apparaît dans Intune. Les paramètres qui sont des liens ouvrir l’inscription à [met à jour de l’optimisation de livraison configurer pour Windows 10](https://docs.microsoft.com/windows/deployment/update/waas-delivery-optimization) dans la documentation de Windows où vous pouvez en savoir plus sur le paramètre.

- **Version de Windows**: la version minimale de Windows 10 qui prend en charge pour ce paramètre.  

- **Détails**: une brève description de la façon dont Intune implémente le paramètre, y compris la valeur par défaut Intune. Lorsqu’il est disponible, il existe des liens vers [fournisseur de service de remise optimisation stratégie configuration](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-deliveryoptimization) les entrées (CSP).  

Pour configurer Intune pour utiliser ces paramètres, consultez [fournir des mises à jour](delivery-optimization-windows.md).  

## <a name="delivery-optimization"></a>Optimisation de la distribution

Ne pas obtenir les mises à jour à partir d’autres ordinateurs sur votre réseau (peer-to-peer). </li> <li> **HTTP fusionné avec un appairage derrière le même NAT (1)**: obtenir les mises à jour à partir d’internet et d’autres ordinateurs sur votre réseau. </li> <li> **HTTP fusionné avec l’homologation entre un groupe privé (2)**: l’homologation produit sur les appareils dans le même Site Active Directory (si elle existe) ou le même domaine. Lorsque cette option est sélectionnée, il croise les adresses IP de la traduction d’adresses réseau (NAT). </li> <li> **Mélange HTTP/peering Internet (3)** : recevez les mises à jour provenant d’Internet et d’autres ordinateurs présents sur réseau. </li> <li> **Mode de téléchargement simple sans peering (99)** : recevez les mises à jour provenant d’Internet et émises par le propriétaire des mises à jour, par exemple, Microsoft. Utiliser un GUID en tant que l’ID de groupe si vous avez besoin créer un groupe unique pour l’homologation de réseau Local pour les branches qui se trouvent sur des domaines différents ou ne sont pas sur le même réseau local.<br><br>**Par défaut**: non configuré <br><br>La stratégie CSP : [DOGroupId](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-deliveryoptimization#deliveryoptimization-dogroupid)     |
